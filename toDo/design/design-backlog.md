## Раздел B — UI компоненты (описаны в дизайн-системе, не реализованы)

### B1. Unit Switcher (01_unit_switcher)
Переключатель единиц измерения: **кг / м / шт**. Три варианта:
- Segmented Control (pill-style) для 2-3 опций
- Dropdown с иконками
- Input Integration (аддон в поле ввода)

Используется в: карточках товаров, формах списания, фильтрах склада.

---

### B2. Row Grouping / Batch Indicator (07_row_grouping)
Иерархия строк в таблицах: **Родительский лист → Обрезки**.
- Tree line (синяя вертикальная линия + L-branch)
- Chevron для expand/collapse
- Batch badges: Available (зелёный), Reserved (красный), Offcut (оранжевый)
- Дочерние строки на светло-сером фоне

Используется в инвентаризационных таблицах.

---

### B3. Scanner Interface (14_scanner_interface)
Полноэкранный интерфейс сканирования QR/штрих-кода:
- Зона фокусировки с синими brackets
- Лазерная линия (анимация)
- Режимы: QR, barcode, серийный номер
- Tooltip при обнаружении кода
- Кнопка ручного ввода

Используется при приёмке товара (2.2) и на мобильных терминалах склада.

---

### B4. Sheet Visualizer (19_sheet_visualizer)
Интерактивная карта раскроя листа металла:
- Цветовое отображение занятых/свободных/обрезочных зон
- Hover → tooltip с размерами куска
- Замок на зарезервированных частях
- Export в DXF/PDF
- Показатель утилизации материала (например, 91.5%)

---

### B5. Sheet Detail View (20_sheet_detail_view)
Карточка конкретной единицы ТМЦ:
- 2-колоночный layout (sidebar 300px + content)
- Timeline событий (history of movement)
- Action sidebar: «Создать раскрой», «Переместить», «Печать этикетки»
- Таблица связанных заказов
- Фото листа + QR-код

---

### B6. Advanced Filter Panel (21_advanced_filters)
Многоуровневая панель фильтрации по физическим параметрам:
- Range slider для толщины (например, 2mm–12mm)
- Material pills (toggle: Active = Blue, Inactive = Gray)
- Warehouse Tree (иерархический выбор склада)
- Числовые поля min/max
- FAB кнопка «Применить фильтры»

---

### B7. Production Stepper (18_production_stepper)
Степпер жизненного цикла заказа:
- Горизонтальный: Received → Processing → Ready → Shipped
- Состояния: Completed (зелёный круг), Active (синий пульс), Pending (серый)
- Прогресс-бар с % выполнения и счётчиком листов
- Компактная версия для ячеек таблицы

---

### B8. Bulk Actions Bar (25_bulk_actions)
Фиксированная панель массовых действий:
- Появляется снизу по центру при выборе ≥ 1 строки
- Тёмный фон `#1F2937`, radius 12px, box-shadow
- Счётчик «3 items selected»
- Кнопки: Scrap (деструктивная), Move, Edit, Print labels

## Раздел C — UX-паттерны (описаны в дизайне, не реализованы)

### C1. Login как overlay поверх Landing
Дизайн: клик на «Войти» плавно трансформирует central card в форму логина без перехода на другую страницу. Код: отдельный маршрут `/login`.

### C2. Register Step 2 как full-screen форма
Дизайн: после заполнения Step 1 (popup) → новый экран без хедера и меню с полями компании. Код: единая страница `/register`.

### C3. Paywall popup для заблокированных функций
Дизайн (`00.2_Registration.md`): при клике на заблокированные функции — попап «Функция доступна в тарифе [Premium]. [Подключить подписку]», элементы становятся серыми с иконкой замка. Не реализовано.

### C4. Column Constructor в таблицах
Документация (`04.1_Suppliers_List.md`, блок 7): возможность скрывать и переупорядочивать колонки таблицы через drag-and-drop. Не реализовано.

### ~~C5. «Сохранить текущие фильтры как вид»~~ — реализовано
`saveView()` + `loadPrefs()` + кнопка в `SuppliersListPage.vue:366`. Сохраняет `activeView` и все фильтры в `localStorage('suppliers_list_prefs')`. Восстанавливается на `onMounted`.

### C6. User State Persistence — реализовано частично
Реализовано только для `SuppliersListPage` (`activeView` + фильтры → `suppliers_list_prefs`). **Не реализовано:** сортировка и набор колонок даже для SuppliersListPage; все остальные страницы (`ProductsPage`, аналитика и т.д.) не сохраняют состояние.

### ~~C7. Экспорт XLSX/CSV списка поставщиков~~ — реализовано
`exportCsv()` в `SuppliersListPage.vue:200` — полная реализация: строит CSV из `suppliers.value`, создаёт `Blob`, скачивает `suppliers.csv`. Кнопка в шаблоне с `v-if="showExport"` (флаг `supplierExport = true`).

### C8. Тултипы на иконки статусов в таблицах — реализовано частично
`v-tooltip` зарегистрирован глобально в `main.js`. Используется в `SuppliersListPage.vue` на иконках в таблице (строки 464, 489, 506) и в `KanbanCard.vue`. **Не реализовано:** в остальных таблицах системы (аналитика, BCC, дефицит — без тултипов на статусные иконки).
