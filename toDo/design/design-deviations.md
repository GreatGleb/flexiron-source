## 2. Цветовая палитра

Документация указывает одни значения, в коде — свои. **В коде используется Ant Design / ANT-based палитра, а не Tailwind (которую описывают доки).** Нужно зафиксировать именно кодовые значения как официальные.

| Токен | В документации | В коде | Где в коде |
|---|---|---|---|
| Success Green | `#10B981` (Tailwind emerald) | `#52c41a` (Ant Design green) | `_status-pills.css`, bar chart |
| Danger Red | `#EF4444` (Tailwind red) | `#ff4d4f` (Ant Design red) | `--danger`, `_status-pills.css` |
| Warning Yellow | `#F59E0B` (Tailwind amber) | `#faad14` (Ant Design gold) | `_status-pills.css` |
| Glass border opacity | `rgba(255,255,255,0.25)` | `rgba(255,255,255,0.18)` | `--glass-border` |

## 4. Sidebar — структура и навигация

**Дизайн-доки описывают flat-список без группировок.** В коде — один `nav-group-label` с лейблом **Modules**. Структура (`AdminSidebar.vue`):
1. Dashboard → `/admin/analytics/dashboard` (без лейбла)
2. — **Modules** (лейбл) —
3. Products → `admin-products`
4. Warehouse → `#` (заглушка)
5. Sales → `#` (заглушка)
6. Suppliers → `/admin/suppliers`
7. Finance → `#` (заглушка)

Порядок разделов также другой:
- Документация (Sitemap): Products → Warehouse → CRM → Supplying → Finance → Analytics
- Код: Dashboard/Analytics → Products → Warehouse → Sales → Suppliers → Finance (Finance, CRM, Warehouse — заглушки)

## 5. Sidebar — user profile

**Конфликт в документации:** `05.2_Sidebar.md` прямо говорит «NO user profile in sidebar», а `05.0_Dashboard_Layout.md` и промпт `3 sidebar.md` описывают user profile в подвале сайдбара.

**В коде:** user profile есть в обоих компонентах — `AdminTopbar.vue` и `AdminSidebar.vue`. Видимость управляется CSS responsive:
- Десктоп (>1024px): показывается в **topbar**, в sidebar скрыт (`display: none`)
- Мобайл (≤1024px): показывается в **sidebar**, в topbar скрыт (`display: none`)

Это совпадает с `05.2` на десктопе, но реализует `05.0` на мобайле.

## 12. Структура аналитики vs. функциональных разделов

Документация разделяет операционные модули (Склад, Продажи, Поставки) от аналитики (раздел 7). В коде всё аналитическое вынесено в одну группу `/admin/analytics/*`, куда входят страницы по всем разделам (включая дефицит — который документация относит к Складу 2.3).

---

## 13. Feature Flags система

В дизайн-документации не упоминается система feature flags. В коде реализована с **24 флагами** (16 page-level + 8 section-level), управляемыми через `localStorage`. Нужно добавить в техническую документацию.
