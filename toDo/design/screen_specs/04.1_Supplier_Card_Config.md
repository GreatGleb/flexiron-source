# 04.1-Config: Zero-Code Configuration (Supplier Card)

*Source: Algorithm Block 0, 6 | CRM ToDo Section 6 | ZeroCode&BackEnd parts*

---

## 1. Concept & Purpose

Zero-code admin page for configuring the Supplier Card structure. Administrator can create/edit/delete fields, group them into sections, reorder via drag-and-drop, and configure permissions per role.

**Target user:** Administrator only.

---

## 2. UI Layout (2-Column + Bottom Panel)

```
+------------------------+----------------------------------------+
|                        |                                        |
|   FIELD LIBRARY        |        SECTION BUILDER                 |
|   (Global, ~25%)       |        (Entity-specific, ~75%)         |
|                        |                                        |
+------------------------+----------------------------------------+
|                     FIELD/SECTION EDITOR (Bottom, ~25%)            |
+-------------------------------------------------------------------+
```

### 2.1 Left Panel: Global Field Library (~25%)

**Header:** "Global Fields" + search input + "New Field" button

**List items (draggable):**
- Field icon (type indicator)
- Field name (i18n key)
- Usage badge: "Used in 3 entities"
- Drag handle

**Field type icons:**
| Type | Icon | Color |
|------|------|-------|
| Text | Aa | Blue |
| Number | 123 | Green |
| Boolean | Toggle | Purple |
| Email | @ | Cyan |
| Date | Calendar | Orange |
| Money | $ | Gold |
| Enum | List | Pink |
| Reference | Link | Indigo |
| File | Paperclip | Gray |

### 2.2 Right Panel: Section Builder (~75%)

**Header:** "Supplier Card Structure" + "Add Section" button

**Section cards (glass-panel style):**
```
+------------------------------------------+
| [drag] SECTION NAME          [edit] [del]|
|------------------------------------------|
| Permissions: Admin: full, Sales: read    |
|------------------------------------------|
| [drag] Field 1    [drag] Field 2         |
| [drag] Field 3    [+ Add from Library]   |
+------------------------------------------+
```

**Actions per section:**
- Drag to reorder
- Edit section properties (name, permissions)
- Delete section (with confirmation)
- Collapse/expand

**Actions per field:**
- Drag to reorder within section
- Remove from section (returns to library)
- Click to edit properties (opens in bottom panel)

### 2.3 Bottom Panel: Field/Section Editor (~25%)

**Contextual panel - shows when field/section selected.**

**Section Editor:**
- Name input (with i18n)
- Permissions matrix (4 actions × 4 roles)
- Description/tooltip text

**Field Editor:**
- Field name (i18n)
- Field type (readonly - from library)
- Required toggle
- Default value
- Validation rules (regex, min/max)
- Permissions matrix (override section defaults)
- Description/tooltip

## 3. Permissions Matrix (ACL)

**Both levels:** Section sets defaults, fields can override.

### 3.1 Actions
| Action | Description |
|--------|-------------|
| **Read** | View field value |
| **Edit** | Modify existing value |
| **Create** | Set value on new entity |
| **Delete** | Clear/remove value |

### 3.2 Roles
| Role | Typical Access |
|------|----------------|
| **Admin** | Full access to all |
| **Sales** | Read/Edit on commercial fields |
| **Warehouse** | Read on logistics fields |
| **Accounting** | Read/Edit on financial fields |

### 3.3 UI Pattern
```
+----------+------+------+--------+--------+
| Role     | Read | Edit | Create | Delete |
+----------+------+------+--------+--------+
| Admin    | [x]  | [x]  | [x]    | [x]    |
| Sales    | [x]  | [ ]  | [ ]    | [ ]    |
| Warehouse| [x]  | [ ]  | [ ]    | [ ]    |
| Account. | [x]  | [x]  | [x]    | [ ]    |
+----------+------+------+--------+--------+
```

**Inheritance indicator:** "Inherits from section" badge (click to override)

## 4. Field Types (from Global Library)

### 4.1 Simple Types
- **Text** - single/multi-line, regex validation
- **Number** - integer/decimal, min/max range
- **Boolean** - checkbox/toggle
- **Email** - validated format, clickable
- **Color Picker** - HEX value, visual swatch

### 4.2 Temporal Types
- **Date** - datepicker
- **Date Range** - two dates as single object
- **Timestamp** - datetime with timezone

### 4.3 Financial Types
- **Money** - number + currency selector
- **Quantity with UOM** - number + unit selector (kg, m, pcs)

### 4.4 Structural Types
- **Enum** - dropdown with configurable options
- **Reference** - link to another entity (Supplier, Client, Product)
- **External Link** - URL with validation

### 4.5 Media Types
- **File** - single file upload
- **Photo** - image with preview
- **Document** - PDF/Office with versioning

### 4.6 Calculated Types
- **Formula** - auto-calculated from other fields (read-only in card)

## 6. Visual Design

### 6.1 Style
- **Glassmorphism:** `backdrop-filter: blur(12px)`, `rgba(255,255,255,0.05)`
- **Drag state:** Dashed border, semi-transparent
- **Drop zone:** Highlighted with accent color
- **Selected state:** Primary blue glow

### 6.2 Color Coding
| Element | Color |
|---------|-------|
| System fields (non-removable) | Muted gray badge |
| Custom fields | Primary blue badge |
| Required fields | Red asterisk |
| Permission override | Orange indicator |

---

## 7. Technical Requirements

### 7.1 Core
- **i18n:** All labels/tooltips with `data-i18n` keys (RU, LT, EN)
- **Audit:** Every config change logged (who, when, what)
- **Validation:** Before save - check required fields, unique names
- **Persistence:** Config stored per entity type

### 7.2 Responsive
- **Desktop (>1200px):** 2-column top + bottom panel
- **Tablet (768-1200px):** Library collapsible, Builder full-width, Editor bottom
- **Mobile (<768px):** Tab navigation (Library | Builder | Editor)

### 7.3 Drag-and-Drop
- Use native HTML5 drag API or library
- Visual feedback during drag
- Undo action (Ctrl+Z support)

## 9. Default Configuration

Supplier Card ships with default sections:

| Section | Fields | Default Permissions |
|---------|--------|-------------------|
| **Status & Requisites** | Status, Company Name, VAT, Address, Contact | Admin: full, Sales: read/edit |
| **Procurement Settings** | BCC Emails, Specialization, Currency, Lead Time | Admin: full, Sales: read |
| **Logistics** | Min Order, Payment Terms | Admin: full, Warehouse: read |
| **Activity Feed** | Stock Snapshot, Request History | Admin: full, Sales: read |
| **Files** | Attachments | Admin: full, All: read/create |
| **Zero-Code Properties** | (dynamic) | Admin: full |
