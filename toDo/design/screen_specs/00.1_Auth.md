# ТЗ на дизайн: 00.1 Authorization (Вход в систему)

## Цель (Goal)
Обеспечить быстрый и безопасный доступ к системе для уже зарегистрированных пользователей.

---

## 1. Визуальное решение (Pop-up)
*   **Тип:** Всплывающее модальное окно (Pop-up), плавно появляющееся поверх Лендинга.
*   **Контейнер:** Округлая Glass-карточка с сильным размытием фона системы. 

## 2. Поля ввода (Inputs)
*   **Логин:** Одно поле с поддержкой трех форматов (Никнейм / Email / Телефон).
    *   *Плейсхолдер:* «Никнейм, Email или Телефон».
*   **Пароль:** Поле с иконкой «глаз» для переключения видимости.
    *   *Функция:* «Забыли пароль?» под полем.

## 3. Элементы управления
*   **Кнопка «Войти»:** Основное действие (Blue Glass style).
*   **Чекбокс «Запомнить меня»:** Для сохранения сессии.
*   **Переход:** Ссылка «Нет аккаунта? Зарегистрироваться» внизу формы.

## 4. Обработка ошибок (Validation)
*   Подсветка поля красным неоновым цветом при неверном вводе.
*   Тултип с подсказкой: «Пользователь не найден» или «Неверный пароль».
