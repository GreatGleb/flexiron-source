## Текущее состояние (анализ кода)

### 1. SuppliersListPage.vue — ✅ ПОЛНОСТЬЮ РЕАЛИЗОВАНА

**Что есть:**
- Заголовок страницы + хедер с кнопками (New Supplier, BCC Request, Export CSV)
- ViewTabs переключение Table / Kanban (под feature flag `supplierKanbanView`)
- Фильтры: статус (CustomSelect), категория (MultiSelect), рейтинг (RatingSelect)
- Save View (localStorage) — сохраняет activeView + все фильтры
- Таблица: Company (с поиском inline), Status (pill), Rating (звёзды), Specialization (tags), Lead Time, Last Request, Contacts (email), Actions (view + BCC)
- Pagination с page size selector (25/50/100) + page numbers
- Kanban view: 6 колонок по статусам, drag-and-drop с confirm-модалом
- Состояния: Loading (skeleton), Empty, Error
- Дефицит-индикатор (красная точка) для поставщиков с `hasDeficit`
- Экспорт CSV (feature flag `supplierExport`)
- Поиск с debounce 300ms

**Что отсутствует (по спецификации):**
- Нет фильтра по динамическим zero-code полям (спецификация упоминает "dynamic zero-code fields" в фильтрах)
- Нет кнопки "Reset Filters" (в спецификации не указана явно, но есть в других страницах)
- Нет тестов (e2e)

### 2. SupplierCardPage.vue — ✅ РЕАЛИЗОВАНА

**Что есть:**
- 3-колоночный layout через `SupplierFormSections` (zero-code динамические секции)
- Action bar: BCC Tool link, Config link, Save button (dirty state)
- Правая колонка: Pricing History (таблица), Files (DropZone + FileItem)
- Audit Log (полная ширина): таблица с user avatar, diff (old→new), delete с confirm-модалом
- Save bar: всегда видна, disabled пока не dirty, loading state
- Skeleton loading для всех 3 колонок
- `useDirtyCheck` composable для отслеживания изменений
- PATCH с дельтой (clean-slate save mode)

**Что отсутствует (по спецификации):**
- Activity Feed (правая колонка, 25%) — в спецификации указана как третья колонка, но в коде правая колонка содержит Pricing History + Files. Activity Feed не реализован отдельно.
- Валидация required полей с red glow (спецификация: "Validation: required fields with red glow") — не видно в коде
- Нет тестов (e2e)

### 3. SupplierCreatePage.vue — ✅ СУЩЕСТВУЕТ (нужно проверить содержимое)

### 4. BccRequestPage.vue — ✅ ПОЛНОСТЬЮ РЕАЛИЗОВАНА (1350 строк)

**Что есть:**
- Products selection: grouped by category, search, category filter, pagination, select all
- Recipients selection: search, pagination, select all/deselect all, sorted (selected float to top)
- Email template editor (EmailTemplate component): subject, body, attachments (DropZone)
- Split button: Send (primary) + Log (dropdown with source options: BCC Tool, Email, Phone, Messenger, Other)
- History table: date, request ID, supplier, product, source, status (sent/responded/no_response)
- Accept Response modal: price input with unit selector
- Mark No Response action
- Auto-rebuild email body on product selection change
- Preselect supplier via `?supplier=` query param
- Idempotency key for send/log requests

**Что отсутствует (по спецификации 04.2-bcc-request-tool.md):**
- Нужно сверить со спецификацией BCC Request Tool

### 5. SupplierCardConfigPage.vue — ✅ ПОЛНОСТЬЮ РЕАЛИЗОВАНА (1053 строки)

**Что есть:**
- Global Field Library (левая колонка): search, list of fields, create new field modal
- Section Builder (правая колонка): sections with drag-drop reorder, collapse/expand, visibility toggle, rename
- Add field to section, remove field from section
- Permissions Matrix (feature flag `permissionsEditor`): 4 actions × 4 roles, user overrides, cascade rules (section→fields), collapse logic
- CRUD for sections and fields (all local, batch save)
- Save bar with loading state

**Что отсутствует (по спецификации 04.1-supplier-card-config.md):**
- Нет "Reference" и "File" типов полей (в спецификации: Text, Number, Boolean, Email, Date, Money, Enum, Reference, File — в коде: text, number, date, boolean, enum, tags)
- Нет типа Money
- Нет типа Email
- Нет типа Reference
- Нет типа File (хотя FileItem/DropZone есть в карточке)
- Нет тестов (e2e)
