## Задачи, отложенные на потом (после обновления заказов)

1. **История заказов в карточке клиента** — данные `orderHistory` уже приходят с сервера, нужно только добавить секцию в `ClientCardPage.vue`.

2. **Аналитика отказов / Internal Notes & History** — по аналогии с карточкой поставщика (секция Internal Notes & History). Нужно хранить и отображать заметки менеджера, причины отказов, историю взаимодействий.

3. **Валидация email и companyCode при создании клиента** — в `ClientCreatePage.vue` валидируется только `name`, хотя API требует `email` и `companyCode`.

4. **Динамические свойства клиента** — конструктор полей (раздел 6 ТЗ), не реализован.

## Резюме: Ключевые расхождения (с учётом приоритетов)

1. **Связь Заказа со Складом** — backend-задача, не влияет на текущий фронтенд.

2. **Генерация документов** — по ТЗ критическая функция. Сейчас только заглушка.

3. **Гибкий конструктор свойств** (раздел 6 ТЗ) — не реализован ни для клиентов, ни для заказов.

4. **Inline-редактирование товаров** в карточке заказа — API поддерживает, UI отсутствует.

5. **Канбан-доска** для заказов — отключена фича-флагом.

6. **Отображение финансовых показателей** (НДС, скидки, маржинальность услуг) — данные есть, UI не показывает.
