## Bugs

### БАГ-1 — HTML-комментарии внутри `<template>` (Pitfall #9)

**File:** `frontend_vue/src/views/admin/orders/OrdersListPage.vue`, `OrderCardPage.vue`, `OrderCreatePage.vue`, `AddOrderItemsModal.vue`, `AddOrderServicesModal.vue`
**Severity:** Medium — комментарии попадают в DOM, увеличивают размер HTML

#### Problem

Во всех view-файлах заказов есть HTML-комментарии `<!-- -->` внутри `<template>`. По правилу Pitfall #9, комментарии в `<template>` запрещены — они попадают в итоговый DOM. Комментарии должны быть только в `<script>`.

Найдено 25+ инстансов.

#### Fix

Удалить все `<!-- -->` из `<template>` во всех view-файлах orders. Комментарии внутри шаблонов использовать нельзя.

---

### БАГ-2 — Cross-namespace i18n: `t('warehouse.unit_*')`

**File:** `frontend_vue/src/views/admin/orders/AddOrderItemsModal.vue:45,125`, `AddOrderServicesModal.vue:44`, `OrderCardPage.vue`, `OrderCreatePage.vue`
**Severity:** Low — uses warehouse domain keys from orders domain

#### Problem

Функция `t('warehouse.unit_X')` вызывается из кода домена orders. Это кросс-доменная ссылка. Если warehouse-ключи изменятся, orders сломается.

#### Fix

Добавить ключи `orders.unit_*` в `orders.ts` и использовать их вместо `t('warehouse.unit_*')`.

---

### БАГ-3 — Undefined CSS class `btn-ghost`

**File:** `frontend_vue/src/views/admin/orders/OrderCardPage.vue:289`
**Severity:** Low — `btn-danger` provides styling but intended `btn-ghost` modifier does nothing

#### Problem

The delete button uses `class="btn btn-ghost btn-danger"`. The class `btn-ghost` is not defined in any accessible CSS file.

#### Fix

Remove `btn-ghost` from the class list, or add `.btn-ghost` styles.

---

### БАГ-4 — `batch-notes-input` class not accessible from OrderCardPage

**File:** `frontend_vue/src/views/admin/orders/OrderCardPage.vue:478`
**Severity:** Medium — notes textarea missing batch-notes-specific resize behavior

#### Problem

The notes textarea uses `class="glass-input batch-notes-input"`. The `.batch-notes-input` class is defined in `warehouse_list.css` (line 705), but OrderCardPage.vue doesn't import that file.

#### Fix

Move `.batch-notes-input` rules from `warehouse_list.css` to `orders_card.css`, or remove the class and use only `glass-input`.

---

### БАГ-5 — Cross-namespace i18n: `t('bcc.selected_text')` in OrderCreatePage

**File:** `frontend_vue/src/views/admin/orders/OrderCreatePage.vue:193`
**Severity:** Low — similar to БАГ-2

#### Problem

`t('bcc.selected_text', 'selected')` is called from OrderCreatePage.vue.

#### Fix

Add a `selected_text` key to `orders.ts` and use `t('orders.selected_text')`.

---

### БАГ-6 — Undefined CSS class `file-list`

**File:** `frontend_vue/src/views/admin/orders/OrderCardPage.vue:607`, `OrderCreatePage.vue:422`
**Severity:** Low

#### Problem

Both pages use `class="file-list"` on file item containers, but `.file-list` is not defined in any CSS file.

#### Fix

Define `.file-list` in a shared CSS or remove the unused class.

---

### БАГ-7 — Undefined i18n key `t('modal.delete_audit_warning')`

**File:** `frontend_vue/src/views/admin/orders/OrderCardPage.vue:724`
**Severity:** High — key not found in any locale, returns literal string at runtime

#### Problem

The audit delete confirmation modal uses `t('modal.delete_audit_warning')`. This key does not exist in any i18n domain. At runtime, `vue-i18n` falls back to returning the key string `'modal.delete_audit_warning'`, which is displayed to the user as raw text.

#### Fix

Add `modal.delete_audit_warning` key to all 3 locales in `orders.ts` (or `common.ts`). The key already exists in the plan but was not implemented.
