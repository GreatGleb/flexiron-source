## 🟡 БАГ-04 — Нельзя очистить поле description через patchService

**Файл:** [`src/services/servicesService.ts`](../../frontend_vue/src/services/servicesService.ts)

**Серьёзность:** Средняя — данные не синхронизируются с UI

### Проблема

```ts
if (data.description) { payload.description = data.description }
```

Проверка на falsy отсекает пустую строку `""`. Если пользователь стирает описание, пустая строка не отправляется на сервер — старое значение остаётся.

### Исправление

Заменить на явную проверку `undefined`:
```ts
if (data.description !== undefined) { payload.description = data.description || null }
```

## 🔴 БАГ-12 — Форма карточки услуги хранит строку вместо TranslatedString

**Файлы:**
- [`src/composables/useServiceCard.ts`](../../frontend_vue/src/composables/useServiceCard.ts)
- [`src/views/admin/products/ServiceCardPage.vue`](../../frontend_vue/src/views/admin/products/ServiceCardPage.vue)
- [`src/services/servicesService.ts`](../../frontend_vue/src/services/servicesService.ts)

**Серьёзность:** Высокая — теряются переводы при сохранении, поля не обновляются при смене языка

### Проблема

`useServiceCard` хранил `form.name` и `form.description` как **простые строки** (`string`), а не как `TranslatedString`. Из-за этого:

1. **Переключение языка не обновляло поля формы** — computed getter не пересчитывался, т.к. нет привязки к `locale`
2. **Сохранение стирало другие локали** — `patchService` оборачивал строку в `toTranslatedString()`, создавая `{ ru: "value", en: "", lt: "" }`
3. **Discard не восстанавливал мультиязычные данные** — извлекал строку только для одной локали

### Исправление

1. **`useServiceCard.ts`** — тип `form.name` и `form.description` изменён с `string` на `TranslatedString | null`. В `load()`, `save()` и `discard()` теперь присваивается весь объект перевода целиком.
2. **`ServiceCardPage.vue`** — добавлены computed свойства `formName` и `formDescription` с getter/setter (аналогично `CategoryCardPage`):
   - **get**: `tf(form.value.name)` — показывает значение для текущей локали
   - **set**: `mergeLocaleValue(form.value.name, v, locale.value)` — обновляет только текущую локаль
3. **`servicesService.ts`** — `patchService` теперь умеет принимать как `string` (для `createService`), так и готовый `TranslatedString` (от формы карточки). Добавлена функция `toPayloadValue()`.

## 🟢 БАГ-13 — Отсутствует CSS-класс `.page-size` — per-page dropdown отображается вертикально

**Файл:** [`src/styles/admin/services_list.css`](../../frontend_vue/src/styles/admin/services_list.css)

**Серьёзность:** Средняя — визуальный баг: dropdown «на странице» отображается вертикально над номером страницы вместо горизонтального расположения

### Проблема

В шаблон `ServicesPage.vue` был добавлен блок `.page-size` с `CustomSelect` для выбора количества записей на странице. Однако CSS-класс `.page-size` (с `display: flex; align-items: center; gap: 8px`) отсутствовал в `services_list.css`. Без этого класса `<div class="page-size">` рендерился как блочный элемент, и dropdown оказывался вертикально над пагинационными кнопками.

На странице товаров (ProductsPage) этот класс был определён — там всё работало корректно.

### Исправление

Добавлен CSS-класс `.page-size` в `services_list.css`:
```css
.page-size {
  display: flex;
  align-items: center;
  gap: 8px;
  white-space: nowrap;
}
```

Позже CSS был рефакторингован — все общие классы пагинации вынесены в [`_pagination.css`](../../frontend_vue/src/styles/admin/components/_pagination.css), а из `services_list.css` дублированный блок удалён.

  text-decoration: none;
