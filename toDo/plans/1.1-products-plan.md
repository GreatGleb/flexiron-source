## ТЗ (выжимка из источников)

### Что такое Товар

Товар — эталонная карточка номенклатурной единицы. Не физическое наличие (это страница «Склад»). У каждого товара — базовые поля (название, цена, единица) и динамические поля, определённые его категорией (и цепочкой родительских категорий).

### Страница списка (ProductsPage — расширить существующую)

Существующий файл `ProductsPage.vue` содержит только хедер и ссылку на категории. Нужно добавить:
- Таблица всех товаров из всех категорий
- Колонки: Название, Категория, Базовая цена, Единица измерения
- Поиск по названию и фильтр по категории (оба внутри GlassPanel — pitfall #19)
- Кнопка «Добавить товар» → модал создания (рядом с кнопкой «Категории» в хедере)
- Клик по строке → `ProductCardPage`
- Удаление через кнопку в строке с confirm-модалом

### Страница карточки (ProductCardPage — новый файл)

**Секция «Основное»:**
- Название (обязательное)
- Категория (CustomSelect, обязательное — менять нельзя после создания; feature flag `productCategoryChange` = false)
- Артикул / SKU (text, опционально)
- Описание (textarea, опционально)

**Секция «Цена»:**
- Базовая цена (число, опционально)
- Единица измерения (CustomSelect): `EUR/vnt` / `EUR/kg` / `EUR/m`

**Секция «Динамические поля»** (из категории + цепочки родителей):
- Все поля категории (inheritedFields + fields объединены в одну редактируемую секцию)
- Ввод зависит от типа поля:
  - `text` → `<input type="text">`
  - `number` → `<input type="number">`
  - `email` → `<input type="email">`
  - `date` → `<input type="date">`
  - `boolean` → чекбокс
  - `enum` → CustomSelect с options из определения поля
  - `file` → DropZone (сразу POST /api/uploads, в payload — fileId)
- Скрыта если у категории нет полей (computed: `allCategoryFields.length === 0`)
- Унаследованные поля (inheritedFields) — помечены иконкой/текстом «Унаследовано»

**Секция «Поставщики»** (section-level flag `productSupplierLinks`):
- Read-only список поставщиков данного товара (name + price + leadDays)
- router-link на карточку поставщика
- Пустое состояние: «Поставщики не привязаны»

**Save bar** (clean-slate):
- Видна при `isDirty`
- Кнопка «Отменить изменения» (`btn_discard_changes`) + «Сохранить»
- Save → PATCH /api/products/:id с дельтой

### API endpoints

```
GET    /api/products              → PaginatedResponse<ProductListItem>  (filters: search, categoryId)
GET    /api/products/:id          → Product (с fieldValues[], linkedSuppliers[])
POST   /api/products              → Product
PATCH  /api/products/:id          → Product
DELETE /api/products/:id          → void
```

### Промпт 7 — Composable: useProductCard (карточка)

```
Задача: создай src/composables/useProductCard.ts.

export function useProductCard(id: string) {
  product: ref<Product | null>(null)
  loading: ref<boolean>(false)
  saving: ref<boolean>(false)
  error: ref<string | null>(null)

  { isDirty, track, diff, reset } = useDirtyCheck<Pick<Product,'name'|'sku'|'description'|'price'|'minStock'|'priceUnit'>>()

  // Локальная копия fieldValues для редактирования
  fieldValues: ref<Record<string, Product['fieldValues'][number]['value']>>({})
  // Исходные fieldValues для сравнения на dirty
  originalFieldValues: ref<string>('')  // JSON-сериализация для сравнения

  fieldValuesChanged: computed<boolean>
    — JSON.stringify(fieldValues.value) !== originalFieldValues.value

  isAnythingDirty: computed<boolean>
    — isDirty.value || fieldValuesChanged.value

  load(): async
    — product.value = await getProduct(id)
    — track({ name, sku, description, price, minStock, priceUnit })
    — fieldValues.value = Object.fromEntries(
        product.value.fieldValues.map(fv => [fv.fieldId, fv.value])
      )
    — originalFieldValues.value = JSON.stringify(fieldValues.value)

  save(): async
    — if (!isAnythingDirty.value) return
    — saving.value = true
    — строим delta:
        const delta: Parameters<typeof patchProduct>[1] = {}
        if (isDirty.value) Object.assign(delta, diff({...product.value}))
        if (fieldValuesChanged.value) {
          delta.fieldValues = product.value!.fieldValues.map(fv => ({
            ...fv,
            value: fieldValues.value[fv.fieldId] ?? null
          }))
        }
    — await patchProduct(id, delta)
    — toast.success(t('products.toast_saved'))
    — await load()  — перезагрузить, reset dirty
    — при ошибке: toast.error(t('products.toast_error'))
    — finally: saving.value = false

  discard(): — await load()  (перезагрузить с сервера)

  return { product, loading, saving, error, fieldValues, isAnythingDirty, load, save, discard }
}

ВАЖНО: product.value.fieldValues — это массив из API. В fieldValues composable
храним плоский Record<fieldId, value> для v-model на каждое поле отдельно.
При PATCH собираем обратно в массив.

После создания: npm run typecheck.
Checkpoint: 0 ошибок.
```
