# Orchestrate — Оркестратор реализации страницы

Знает полный алгоритм и определяет откуда начать по состоянию файлов.
Скилы выполняются inline как обычно — со всеми своими СТОП-ами внутри.
Оркестратор включается только на переходах между скилами.

---

## CRITICAL RULES

1. **Не управлять стопами внутри скилов** — каждый скил сам останавливается когда нужно
2. **После завершения скила — сразу объявить следующий** без запроса у пользователя
3. **Состояние = файлы** — текущий этап определяется только по файлам, не по памяти
4. **Inline выполнение** — скилы запускаются в этом же контексте, не как суб-агенты
5. **Resume без вопросов** — определить этап самостоятельно, объяснить пользователю и продолжить

---

## Шаг 0: Определить Plan ID и текущий этап

### Извлечь Plan ID

Из сообщения пользователя:
- "создай страницу 1.2" → id = "1.2"
- "продолжи products" → Glob `toDo/plans/` → найти файл с "products" → извлечь id

Если не удаётся — спросить один раз: «Укажи ID страницы (например 1.2)»

### Определить текущий этап по файлам

Читать в этом порядке и остановиться на первом совпадении:

```
1. toDo/plans/{id}-*-plan.md НЕ существует
   → Этап: CREATE-PLAN

2. План существует, но src/views/admin/**/*Page.vue для этой страницы НЕ существует
   → Этап: CREATE-PAGE (Phase 0)

3. Vue-файл существует, bugs-файл toDo/plans/bugs/{id}-*-bugs.md НЕ существует
   → Спросить: «Страница создана частично или все Phase 0–9 завершены?»
     - Не завершены → Этап: CREATE-PAGE (продолжить с нужной фазы)
     - Завершены → Этап: PRE-MANUAL-CHECK

4. Bugs-файл существует, есть незафиксированные баги (без ✅)
   → Этап: FIX-BUGS

5. Все баги ✅, тестовый spec-файл tests/e2e/admin/**/{domain}.spec.ts НЕ существует
   → Этап: PHASE-10

6. Spec-файл существует, в bugs-файле есть ✅ баги которые не проходили update-skills
   → Этап: UPDATE-SKILLS

7. Всё выполнено
   → Сообщить: «Страница {id} полностью готова»
```

### Объявить пользователю

```
Страница {id}: [NEW / продолжаю с этапа X]
Начинаю: [название текущего этапа]
```

---

## Алгоритм выполнения

Выполнять этапы по порядку. После завершения каждого скила — **СТОП, ждать подтверждения** перед запуском следующего. Стопы внутри скилов — это их собственные внутренние остановки, оркестратор в них не вмешивается.

Формат СТОП оркестратора (между скилами):
```
⏸ СТОП — [Текущий скил] завершён
[Краткий итог: что создано/исправлено]
Следующий: [название следующего этапа]
Продолжить?
```

---

### Этап CREATE-PLAN

Выполнить скил `.claude/skills/create-plan.md` полностью.

После завершения:
```
⏸ СТОП — create-plan завершён
План: toDo/plans/{id}-*-plan.md
Следующий: create-page Phase 0–9
Продолжить?
```

---

### Этап CREATE-PAGE (Phase 0–9)

Выполнить `.claude/skills/create-page.md` Phase 0–9.

Скил делает свои СТОП-ы после каждой фазы — пользователь подтверждает переходы внутри скила как обычно.

После завершения Phase 9:
```
⏸ СТОП — create-page Phase 0–9 завершён
Следующий: pre-manual-check (20 автоматических проверок)
Продолжить?
```

---

### Этап PRE-MANUAL-CHECK

Выполнить `.claude/skills/pre-manual-check.md` полностью.

Скил делает свои СТОП-ы после каждого из 20 промптов.

После завершения всех 20 проверок:
```
⏸ СТОП — pre-manual-check завершён
Найденные проблемы записаны в bugs-файл.
Следующий: ручное тестирование — протестируй UI и пришли список найденных багов.
```
Ждать сообщения от пользователя со списком багов.

---

### Цикл багов (повторяется до полного закрытия)

Это не одноразовый шаг — цикл повторяется столько раз сколько нужно:

```
LOOP:
  1. Пользователь присылает список багов (любое количество)
     → add-bug автоматически записывает все в bugs-файл
     ⏸ СТОП: "Записано {N} багов. Начинаем исправлять?"

  2. fix-bugs — исправляет по одному за стоп, отмечает ✅

  3. После закрытия всех текущих багов:
     ⏸ СТОП: "Все баги исправлены ✅
              Нашёл новые баги при тестировании?
              → Да, вот список: [...] → вернуться к шагу 1
              → Нет → выходим из цикла, переходим к Phase 10"

REPEAT until пользователь подтверждает "новых багов нет"
```

Цикл может повторяться любое количество раз — 2, 5, 10 итераций — пока UI не будет полностью чист.

---

### Этап PHASE-10

Выполнить `.claude/skills/create-page.md` начиная с Phase 10.

После завершения:
```
⏸ СТОП — Phase 10 завершена
Playwright тесты написаны и зелёные ✅
Следующий: update-skills
Продолжить?
```

### Этап UPDATE-SKILLS

Выполнить `.claude/skills/update-skills.md`.

Скил делает свои СТОП-ы после каждого проанализированного бага.

После завершения всех багов:
```
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
✅ Страница {id} полностью готова
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
create-plan      ✅
create-page      ✅ Phase 0–9
pre-manual-check ✅
fix-bugs         ✅
Phase 10         ✅ Playwright тесты
update-skills    ✅
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
```
