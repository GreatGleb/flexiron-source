## ИДЕЯ-01 — Kanban view на странице Товаров (по категориям)

**Контекст:** На странице 1.1 Товары (справочник) добавить вид Kanban рядом с таблицей.

**Кейс использования:** Когда категорий ~5–10 (базовых), менеджер хочет одним взглядом охватить все категории и первые несколько товаров в каждой — без пролистывания таблицы.

**Механика:**
- Колонки = базовые (root-level) категории
- В каждой колонке — первые N товаров этой категории (превью)
- Без возможности добавлять товары прямо из канбана (read-only view)
- Клик на товар → карточка товара

**Почему отложено:**
- Availability/status у товаров (1.1) по ТЗ не существует — физическое наличие только в разделе Склад (1.2)
- Нет осмысленного Enum-поля для группировки кроме `priceUnit`
- При 10+ категориях с дочерними — иерархия усложняет колонки

**Когда уместно:** после реализации 1.2 Склад, когда появится реальный статус партий.
