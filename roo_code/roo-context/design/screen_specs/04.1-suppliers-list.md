# УЛ 04.1: Спецификация «Список поставщиков» (Suppliers List)

*Источник требований: Алгоритм работы ERP Flexiron — Блок 1.1, 3.1, 6.3, Sitemap 4.1*

## 1. Концепция (Concept)
Центральная таблица управления базой поставщиков. Основная задача — быстрая фильтрация по категориям металла, переключение между форматами отображения и переход к массовым запросам цен (BCC).

---

## 2. Header и действия (Header & Actions)

### Кнопки:
*   **Новый поставщик**: Кнопка `+ Новый поставщик` - ссылка на отдельную страницу создания (временно `#`).
*   **BCC Request**: Кнопка с иконкой - ссылка на `bcc-request.html` (как в карточке).

### Переключение вида:
*   **Вкладки**: `[Таблица] [Канбан]` - переключение между видами.

### Поиск:
*   **Поле поиска**: Инстант-поиск по названию, коду или контакту.

---

## 3. Элементы управления и Виды (Views & Controls)
*   **Два формата отображения (Views)**:
    *   **Таблица (Table)**: Компактный список для работы с данными.
    *   **Канбан (Kanban)**: Карточки, сгруппированные по колонкам-статусам (напр. «Активные», «В проверке»). Поддерживается Drag-and-drop для смены статуса.
*   **Система фильтрации**:
    *   *Базовая*: По специализации (категории металла).
    *   *Расширенная*: Фильтрация по **динамическим Zero-code полям** (напр. по рейтингу, по городу склада).
*   **Сохранение видов**: Кнопка «Сохранить текущие фильтры как вид» (Блок 6.3.5 Алгоритма).

---

## 4. Колонки Канбана (по статусам)
Карточки группируются по текущему статусу:

| Колонка | Статус | Класс бейджа |
|--------|--------|--------------|
| **Активно** | Active | `pill-success` |
| **Предпочитаемые** | Preferred | `pill-info` |
| **Новые** | New | `pill-mint` |
| **В проверке** | Under Review | `pill-warning` |
| **Приостановленные** | Suspended | `pill-suspended` |
| **Заблокированные** | Blocked | `pill-danger` |

*Drag-and-drop*: Перетаскивание карточки из одной колонки в другую = смена статуса.

---

## 5. Табличная часть (Data Grid)
Таблица в стиле Glassmorphism с колонками:

| Колонка | Описание | Источник |
|--------|----------|----------|
| **Компания** | Название с ссылкой на карточку (`card.html`). | Справочник |
| **Статус** | Цветной бейдж (HEX-цвета из настроек 0.7). | Справочник |
| **Рейтинг** | Звездная оценка (1-5) надежности. | Карточка |
| **Специализация** | Теги категорий (Листы, Трубы и т.д.). | Справочник |
| **Lead Time** | Средний срок поставки (дней). | Аналитика (7.4) |
| **Последний запрос** | Дата последней BCC-рассылки. | Логи |
| **Контакты** | Иконка Email (BCC-адрес) и Телефон. | Реквизиты |
| **Действия** | Иконки кнопок: открыть карточку, отправить BCC. | - |

### Действия в строках:
*   **Открыть карточку**: Иконка-ссылка на `card.html?id=XXX`.
*   **Отправить BCC**: Иконка-ссылка на `bcc-request.html?supplier=XXX`.

---

## 6. Состояния (States)

### Состояние загрузки:
*   Skeleton-Loader для таблицы (строки с анимированными серыми блоками).
*   Skeleton для Канбана (плацхолдеры карточек).

### Пустое состояние:
*   Сообщение: "Поставщики не найдены."
*   Кнопка: "Добавить первого поставщика" (ссылка `#`).

### Состояние ошибки:
*   Сообщение: "Ошибка при загрузке данных."
*   Кнопка: "Повторить попытку".

---

## 7. Визуальные акценты и Интерактивы
*   **Индикаторы Дефицита**: Красная точка-алерт у поставщика, если в его категориях есть «Заказы в минус».
*   **Конструктор колонок**: Возможность скрывать лишние колонки или менять их порядок перетаскиванием (Drag-and-drop, Блок 6.2 Алгоритма).
*   **Hover-эффект**: Подсветка стеклянного фона строки при наведении.

---

## 8. Технические детали
*   **User State Persistence (Запоминание состояния)**: Система обязана запоминать текущие настройки интерфейса для каждого пользователя (активный Вид, набор и порядок колонок, активные фильтры и сортировку). При возвращении на страницу экран должен восстанавливаться в том виде, в котором он был оставлен.
*   **i18n**: Полная поддержка RU, LT, EN.
*   **Пагинация**: Выбор количества строк на страницу (25, 50, 100) + кнопка «Загрузить еще» для бесконечного скролла.
*   **Export**: Выгрузка в XLSX/CSV (Блок 8.9 Алгоритма).
