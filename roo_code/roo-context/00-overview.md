## Разделение ответственности: Backend vs Zero-code

### Зона Backend (жёсткий код)
Критические узлы, где цена ошибки высока:

| Модуль | Описание |
|--------|----------|
| **Транзакционный модуль «Распил» (Atraižos)** | Атомарная операция: списать родителя → создать обрезок → скопировать свойства → зафиксировать parent_id |
| **Математическое ядро (UOM Conversion)** | Глобальная функция пересчёта (плотность × размеры = вес) на сервере |
| **Логика резервирования и Таймеры** | Cron-задача: снимать просроченные резервы, уведомлять менеджера |
| **Генератор документов (PDF Engine)** | Excel-шаблоны → Proforma-Invoice, CMR, Lipdukai |
| **Защита данных и Журнал аудита** | Audit Trail — любое изменение пишет лог |
| **Валидатор «Анти-дурак»** | Нельзя отгрузить больше чем есть, создать инвойс без VAT, удалить товар с остатком |

### Зона Zero-code (настройка в админке)
Всё, что касается интерфейса и структуры данных:

| Модуль | Описание |
|--------|----------|
| **Конструктор сущностей (Data Schema)** | Создание новых полей (Email, Цвет, Диаметр) без программиста |
| **Интерфейс карточек и таблиц** | Настройка колонок, группировка по категориям |
| **Управление справочниками** | База клиентов, поставщиков, товаров. Права доступа |
| **Воронка статусов (Workflow)** | Создание статусов, привязка триггеров |
| **Центр уведомлений** | Настройка Push-уведомлений |
| **Фильтры и Аналитика** | Создание «Видов» (Views) для отчётов |

### Матрица: кто что делает

| Ситуация | Zero-code (вы) | Backend (программист) |
|----------|---------------|----------------------|
| Добавить новое свойство металла | Добавляете поле "Твердость" в админке | Ничего |
| Изменить формулу веса | Ничего | Меняет строку в "Математическом ядре" |
| Поменять логотип в счете | Загружаете файл в настройки | Ничего |
| Интегрировать весы | Создаете поле "Вес с весов" | Пишет код связи API весов с полем |
