## TZ Summary (from sources)

### What is an Order

Order (Заказ) — the central sales transaction entity in the CRM. It connects Client → Warehouse stock → Services → Documents. An order manages:
- Header: client selection, document type (Local LT / Export EN), status
- Cart: products from warehouse (with cutting/offcuts mechanism), services from service catalog
- Pricing: flexible pricing, discounts, margin display, total weight calculator
- Documents: Proforma-Invoice, CMR, KPO, Certificates, Lipdukai generation
- Status funnel: customizable status workflow with reservation triggers

### Pages needed

#### 1. Orders List Page (`OrdersListPage.vue`)
- Table of all orders with columns: Order#, Client, Status, Total, Date
- Filter by: status, client search, date range
- Quick-action: create new order (→ `OrderCreatePage.vue`)
- Quick-action: delete order (with confirm modal, check for active shipments)
- Row click → `OrderCardPage.vue`
- Future: Kanban view (drag-drop status change)

#### 2. Order Create Page (`OrderCreatePage.vue`)
- Select client from directory (search + select)
- Set document type: Local (LT) / Export (EN)
- Create → redirect to OrderCardPage

#### 3. Order Card Page (`OrderCardPage.vue`)
**Header section:**
- Client info (name, VAT, address — read-only from Client card)
- Order number, creation date
- Document type (Local/Export)
- Status (CustomSelect with status funnel)

**Cart section (Order Items):**
- Add products from warehouse (search + select product/batch)
- Add services from service catalog
- For each item: quantity, unit price (editable), discount, total
- "Cut" mechanism: specify dimensions, system creates offcut
- Show: total weight, total amount (with/without VAT), margin

**Services section:**
- List of services added to order
- Service name, base cost, sale price, margin per service
- Add/remove services

**Documents section:**
- Generate Proforma-Invoice / CMR / KPO / Certificates / Lipdukai
- Document preview with inline editing before final generation
- List of previously generated documents

**History/Audit section:**
- Audit log of all order changes

**Save bar** (clean-slate):
- Visible when `isDirty`
- Save → PATCH with delta
- Discard → reload from server

### API Endpoints (to be defined in contract)

```
GET    /api/orders              → PaginatedResponse<OrderListItem>  (filters: search, status, clientId, dateFrom, dateTo)
GET    /api/orders/:id          → Order (with items[], services[], documents[], auditLog[])
POST   /api/orders              → Order (create header, empty cart)
PATCH  /api/orders/:id          → Order (delta: status, items, services, notes)
PATCH  /api/orders/:id/status   → Order (quick status change)
DELETE /api/orders/:id          → void (check for active shipments)
POST   /api/orders/:id/items    → OrderItem (add product/service to cart)
DELETE /api/orders/:id/items/:lineId  → void (remove from cart)
```

export function useOrderCard(id: string) {
  order: ref<Order | null>(null)
  loading: ref<boolean>(false)
  saving: ref<boolean>(false)
  error: ref<string | null>(null)

  // Dirty check for header fields only (status, notes, documentType)
  { isDirty, track, diff, reset } = useDirtyCheck<Pick<Order, 'status' | 'notes' | 'documentType'>>()

  load(): async
    — order.value = await getOrder(id)
    — track({ status: order.value.status, notes: order.value.notes, documentType: order.value.documentType })

  save(): async
    — if (!isDirty.value) return
    — saving.value = true
    — const delta = diff(order.value)
    — await patchOrder(id, delta)
    — track({ status: order.value!.status, notes: order.value!.notes, documentType: order.value!.documentType })
    — toast.success(t('orders.toast_saved'))
    — finally: saving.value = false

  discard(): — await load() (re-fetch from server, resets all local state)

  // Quick-action item operations (immediate, not part of dirty check)
  handleAddItem(data: ...): async
    — await addOrderItem(id, data)
    — toast.success(t('orders.toast_item_added'))
    — await load() (refresh order)

  handleUpdateItem(lineId: string, delta: ...): async
    — await updateOrderItem(id, lineId, delta)
    — await load()

  handleDeleteItem(lineId: string): async
    — await deleteOrderItem(id, lineId)
    — toast.success(t('orders.toast_item_removed'))
    — await load()

  // Quick-action service operations
  handleAddService(data: ...): async
  handleDeleteService(serviceId: string): async

  // Document generation placeholder
  handleGenerateDocument(type: string): async
    — toast.info(t('orders.toast_document_generated'))
    — await load()

  // Status change (quick-action)
  handleChangeStatus(status: OrderStatus): async
    — await patchOrderStatus(id, status)
    — toast.success(t('orders.toast_status_changed'))
    — await load()

  return { order, loading, saving, error, isDirty, load, save, discard,
           handleAddItem, handleUpdateItem, handleDeleteItem,
           handleAddService, handleDeleteService,
           handleGenerateDocument, handleChangeStatus }
}
